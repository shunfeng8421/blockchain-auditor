## === Solidity / EVM Rules ===

rule_id: BLK-SOL-001
name: "Reentrancy Attack"
severity: critical
category: security
swc_id: SWC-107
cwe_id: CWE-841
tags: [solidity, evm, reentrancy, defi]
detection:
  methods:
    - "External call before state update"
    - "Missing ReentrancyGuard or checks-effects-interactions pattern"
    - "transfer/send replaced with call{value:}() without protection"
  patterns:
    - "call\s*\{.*value:\s*.*\}\(.*\)"
    - "No modifier and cross-function reentrancy path exists"
confidence: high
severity_weight: 100
fix:
  1. Use OpenZeppelin ReentrancyGuard
  2. Follow checks-effects-interactions pattern
  3. Prefer transfer() (2300 gas) over call{value:}()
known_exploits:
  - "The DAO (2016, $60M)"
  - "Cream Finance (2021, $130M)"
  - "Curve/Vyper reentrancy (2023, $70M)"

rule_id: BLK-SOL-002
name: "Integer Overflow/Underflow"
severity: high
category: security
swc_id: SWC-101
cwe_id: CWE-682
tags: [solidity, evm, arithmetic]
detection:
  methods:
    - "Solidity < 0.8.0 unchecked arithmetic"
    - "unchecked {} block arithmetic"
    - "Missing SafeMath with compiler < 0.8.0"
  patterns:
    - "pragma solidity \^?0\.[4-7]\.\d+"
    - "\bunchecked\s*\{"
confidence: high
severity_weight: 70
fix:
  1. Upgrade to Solidity >= 0.8.0 (built-in overflow check)
  2. Or use OpenZeppelin SafeMath
  3. Add explicit bounds check in unchecked blocks
known_exploits:
  - "BeautyChain (BEC) overflow (2018)"
  - "PoWH Coin underflow (2018)"

rule_id: BLK-SOL-003
name: "Missing Access Control"
severity: critical
category: security
swc_id: SWC-105
cwe_id: CWE-284
tags: [solidity, evm, access-control]
detection:
  methods:
    - "Critical functions missing onlyOwner/onlyRole modifier"
    - "Sensitive functions with public visibility"
    - "Ownable not inherited or not used"
    - "initialize function without initializer modifier"
  patterns:
    - "function (withdraw|setOwner|setAdmin|mint|burn|initialize)\s*\(.*\)\s*public"
confidence: high
severity_weight: 100
fix:
  1. Inherit OpenZeppelin Ownable/AccessControl
  2. Add onlyOwner/onlyRole to critical functions
  3. Use initializer modifier to prevent re-initialization
known_exploits:
  - "Parity Multisig (2017, $150M)"
  - "Wormhole uninitialized (2022, $326M potential)"

## === Rust / Solana / Polkadot Rules ===

rule_id: BLK-RST-001
name: "Missing Account Validation"
severity: critical
category: security
tags: [rust, solana, anchor, account-validation]
detection:
  methods:
    - "AccountInfo.owner not checked"
    - "Anchor #[account(...)] constraints insufficient"
    - "Missing signer check (owner/signer not verified)"
  patterns:
    - "AccountInfo.*owner.*!=|AccountInfo.*owner.*ne"
confidence: high
severity_weight: 100
fix:
  1. Anchor: use #[account(owner = program_id)] constraint
  2. Native: manually check AccountInfo.owner == program_id
known_exploits:
  - "Multiple Solana programs drained due to missing owner check"

---

rule_id: BLK-RST-002
name: "Unchecked CPI Return"
severity: high
category: security
tags: [rust, solana, cpi, cross-program]
detection:
  methods:
    - "invoke/invoke_signed return value not checked"
    - "CPI call failure silently ignored"
  patterns:
    - "invoke\(.*\)\s*;\s*$"
    - "invoke_signed\(.*\)\s*;\s*$"
confidence: medium
severity_weight: 60
fix:
  1. Always check CPI call results: invoke(...)? or .unwrap()
  2. Critical CPI failures should revert entire tx

---

rule_id: BLK-RST-003
name: "Bump Seed Mismatch"
severity: high
category: security
tags: [rust, solana, pda, anchor]
detection:
  methods:
    - "PDA bump seed hardcoded instead of retrieved from chain"
    - "Stored bump inconsistent with actual derive"
  patterns:
    - "find_program_address.*bump|bump\s*=\s*\d+"
confidence: medium
severity_weight: 60
fix:
  1. Store bump in PDA accounts field
  2. Read bump from account during CPI, dont hardcode

rule_id: BLK-RST-004
name: "Arithmetic Overflow (Rust)"
severity: medium
category: security
tags: [rust, solana, near, arithmetic]
detection:
  methods:
    - "Using +, -, *, / instead of checked_/saturating_"
    - "Overflow risk in release mode"
    - "Type cast truncation (as conversion)"
  patterns:
    - "as\s+(u\d+|i\d+)\b"
confidence: medium
severity_weight: 40
fix:
  1. Use checked_add/sub/mul, saturating_add etc.

name: "Consensus Critical Logic Without Test Coverage"
severity: high
category: quality
tags: [go, cosmos, consensus, testing]
detection:
  methods:
    - "consensus/validator module missing unit tests"
    - "State transition functions untested"
  patterns:
    - "consensus|validator|state.*\.go missing _test.go"

  3. Add fuzz testing for boundary conditions

## === Universal Crypto Rules ===

rule_id: BLK-CRYPTO-001
name: "Hardcoded Private Keys / Secrets"
severity: critical
category: security
tags: [all, secrets, key-management]
detection:
  methods:
    - "Source code contains private keys/mnemonics/API keys"
    - "Config files contain unencrypted keys"
  patterns:
    - "(private_key|mnemonic|secret_key|api_key)\s*=\s*[\"\\''']\w+[\"\\''']"
    - "0x[a-fA-F0-9]{64}"
confidence: critical
severity_weight: 100
fix:
  1. Use environment variables or key management service
  2. .env file in .gitignore
  3. Private keys/mnemonics never hardcoded

---

rule_id: BLK-CRYPTO-002
name: "Vulnerable Dependencies"
severity: high
category: security
tags: [all, supply-chain, dependencies]
detection:
  methods:
    - "package.json/Cargo.toml/go.mod has deps with known CVEs"
    - "Dependencies >1 year outdated"
  patterns:
    - "Detected via npm audit / cargo audit / go vulncheck"
confidence: medium
severity_weight: 50
fix:
  1. Regular dependency vulnerability scans (CI integration)
  2. Use Dependabot/Renovate for auto-updates
  3. Minimize dependency count
